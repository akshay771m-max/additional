Portfolio Framework — Homepage Scroll Layout

Annotated layout patterns
Full page scroll · top → bottom
① Hero — Type as Texture
Massive "PORTFOLIO" word repeated in a grid IS the background. No image. Typography becomes the visual environment. Bauhaus tactic — form serves function.
② Nav — Pill toggle UX
"Akshay | PORTFOLIO" pill switches identity context. Hamburger opens full-screen overlay menu. Smart: primary CTA (WORKS) floats in viewport center.
③ About — Name as Statement
"AKSHAY KUMAR." at 300%+ scale. First name in white, surname in red. The period makes it a declaration, not just a label. Red illustrated avatar bleeds into dark bg.

Works Grid — Masonry Logic

→ 3-column asymmetric — Col 1 has one tall card spanning full height. Cols 2+3 split into top+bottom pairs. Creates visual weight on left.
→ Numbered badges 01–08 at top-right corner. Each card has a coloured 2px accent line + ↗ link arrow. Minimal info: title + subtitle only.
→ Per-project colour code — Red for Product Design, Blue for Form Sketches, Green for Theme Sketches, Amber for NightOwl, Purple for Posters.
→ Custom cursor appears throughout the works grid — hollow circle with dot, reinforces interactive feel without hover states.
④ Footer — Zero noise
"Designed & built with intention." — The footer is a manifesto close, not just a legal row. Social links in one flat line. Phone in red for visual break.

Site IA — Single-Page Anchor Flow

5 Intentional Choices — Why They Work

① Type-as-hero background — eliminates need for photography. The word "PORTFOLIO" repeated IS the brand statement. Bauhaus: function IS form. No stock images = 100% original.
② Floating "WORKS" pill nav — always in viewport center during scroll. Shortcut to the most important section. Reduces friction for recruiters who want to jump straight to projects.
③ Masonry grid (not uniform cards) — asymmetric heights create visual rhythm. Card 01 (Product Design) is taller = more important. Hierarchy via scale, not just placement.
④ Per-project accent colour — each project has its own colour identity (red/blue/green/amber/purple). Creates instant visual memory anchor. Visitor associates colour with project.
⑤ Illustrated avatar vs photo — red-tinted illustration over dark bg creates brand character without vulnerability of real photo. Manga/comic aesthetic matches design sensibility.

How This Portfolio Was Made

① Layout Diagrams
Hand-drawn layout diagrams mapped the structure — section order, grid logic and spatial hierarchy sketched on paper first.
→
② Design
Iterative design passes refined typography, colour, motion and micro-interactions until every detail felt intentional.
→
③ Wireframes
Lo-fi wireframes were made to validate spacing, component placement and navigation flow before any styling was applied.
→
④ Backend Coding
Pure HTML/CSS was written — animations, SVG filters, scroll logic and print-ready layout rules coded from scratch.

Cursor Engine & Small UI Elements

Hollow outer ring with solid red core. The ring expands & turns red on hover, acting as a dynamic focus lens rather than a static pointer.
